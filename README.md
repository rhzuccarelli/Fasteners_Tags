# Fastener Tracker

A full-stack web application for managing hardware fasteners, organising them into physical storage boxes, and printing precise 50×15mm PDF label tags.

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React 18 + Vite + TailwindCSS |
| Backend | Node.js + Express |
| Database | SQLite via better-sqlite3 |
| PDF generation | Python + ReportLab |
| DXF/PDF conversion | Python + ezdxf + pdf2image |

## Setup

### 1. Install Node dependencies

```bash
npm install
cd client && npm install && cd ..
```

### 2. Install Python dependencies

```bash
pip3 install reportlab ezdxf pdf2image pillow --break-system-packages
```

> On Debian/Ubuntu systems with Python 3.11+, `--break-system-packages` may be needed. Alternatively use a venv.

Poppler is required by pdf2image for PDF → PNG conversion:

```bash
# Ubuntu/Debian
sudo apt install poppler-utils

# macOS
brew install poppler
```

### 3. Run

```bash
npm run dev
```

This starts:
- **Express API** on `http://localhost:3001`
- **Vite dev server** on `http://localhost:5173`

Open `http://localhost:5173` in your browser.

## Features

### Standards Library (`/standards`)
- Pre-seeded with DIN 912, DIN 7991, ISO 7046, DIN 7051, DIN 7997, ISO 4035, DIN 6796
- Upload DXF, PDF, or PNG drawings per standard — auto-converted to PNG thumbnails
- Add / edit / delete custom standards

### Fastener Library (`/fasteners`)
- Full CRUD with filter by metric, standard, and tool type
- Tool type badges (Hex, Phillips, Torx, Flat, Nut, Washer, Other)
- Out-of-stock warning when quantity = 0
- Checkbox selection for bulk PDF export

### Box Organiser (`/boxes`)
- Small (270×110mm) and Large (360×150mm) box sizes
- 1–4 slot divisions per box with SVG slot visualiser
- Click any slot to assign/reassign a fastener via searchable dropdown
- "Suggest Grouping" — auto-assigns fasteners by metric + standard across all boxes

### Print Tags (`/print`)
- **Single fastener tags** (50×15mm): drawing thumbnail · metric · standard · tool type · quantity · cut marks
- **Box tags** (50×15mm): black header with box name + per-slot fastener info in equal columns
- Export selected or all tags as an A4 PDF (3-up grid, 5mm gaps, 10mm margins)
- Tags generated by `scripts/generate_tags.py` using ReportLab

## API

```
GET    /api/standards
POST   /api/standards
PUT    /api/standards/:id
DELETE /api/standards/:id
POST   /api/standards/:id/drawing    # multipart, field: "drawing"

GET    /api/fasteners?metric=&standard_id=&tool_type=
POST   /api/fasteners
PUT    /api/fasteners/:id
DELETE /api/fasteners/:id

GET    /api/boxes
POST   /api/boxes
PUT    /api/boxes/:id
DELETE /api/boxes/:id
PUT    /api/boxes/:boxId/slots/:slotIndex  # body: { fastener_id }

POST   /api/export/tags  # body: { type, ids } → PDF stream
```

## Uploading your first drawing

1. Go to **Standards**
2. Click **Upload Drawing** on any standard card
3. Select a `.dxf`, `.pdf`, or `.png` file
4. The server converts it to PNG (via ezdxf or pdf2image) and stores it in `uploads/`

## File structure

```
fastener-tracker/
├── client/               Vite + React frontend
│   └── src/
│       ├── components/   StandardCard, FastenerCard, BoxCard,
│       │                 BoxSlotVisualiser, TagPreview, UploadDrawing
│       └── pages/        Standards, Fasteners, Boxes, PrintTags
├── server/
│   ├── index.js          Express entry point
│   ├── db.js             SQLite schema + seed data
│   └── routes/           standards, fasteners, boxes, export
├── scripts/
│   ├── generate_tags.py  ReportLab PDF generator
│   └── convert_drawing.py DXF/PDF/image → PNG converter
├── uploads/              Generated PNG thumbnails
└── server/fasteners.db   SQLite database (auto-created)
```

## Database

SQLite file is created automatically at `server/fasteners.db` on first run. Standards are seeded with `INSERT OR IGNORE` so re-runs are idempotent.
